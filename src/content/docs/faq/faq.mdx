## 基础问题

### anssl 是免费的吗？

是的，anssl 的核心功能完全免费，包括：

- 无限数量的证书申请
- 自动续期功能
- 域名监控
- 基础的自动部署功能

我们支持 Let's Encrypt、ZeroSSL、Google Trust Services 等提供免费 DV 证书的 CA 机构。

### anssl 和其他证书管理工具有什么区别？

**与 Certbot 相比**：

- anssl 提供可视化 Web 界面，无需命令行操作
- 支持多服务器统一管理
- 提供证书监控和到期提醒
- 支持自动部署到多种平台

**与传统 CA 官网相比**：

- 自动化程度更高，支持自动续期
- 统一管理多个域名的证书
- 提供批量操作功能
- 更好的用户体验

## 证书申请

### 申请证书需要多长时间？

通常在 1-5 分钟内即可完成证书颁发，具体时间取决于：

- 域名验证方式（DNS 验证可能需要等待 DNS 记录生效）
- CA 机构的处理速度
- 网络状况

### 一个账号可以申请多少张证书？

没有数量限制。您可以为任意数量的域名申请证书。

### 申请失败了怎么办？

常见的申请失败原因：

1. **域名验证未通过** - 检查验证记录是否正确配置
2. **DNS 记录未生效** - 等待 5-30 分钟后重试
3. **域名不可访问** - 确保域名解析正确
4. **CA 机构限流** - Let's Encrypt 有速率限制，可切换到其他 CA
5. **域名不符合规范** - 检查域名格式是否正确

可以在证书详情页查看详细的错误信息。

## 域名验证

### 推荐使用哪种验证方式？

**DNS 验证（推荐）**：

- ✅ 适用于所有证书类型
- ✅ 支持通配符证书
- ✅ 不需要开放 80/443 端口
- ❌ 需要等待 DNS 记录生效

**HTTP 验证**：

- ✅ 验证速度快
- ✅ 配置简单
- ❌ 不支持通配符证书
- ❌ 需要开放 80 端口

## 证书续期

### 证书什么时候会自动续期？

开启自动续期后，系统会在证书到期前 20 天开始尝试续期。

续期成功后，如果配置了自动部署，系统会自动将新证书部署到服务器。

### 自动续期失败了怎么办？

系统会在续期失败时发送通知邮件。常见失败原因：

1. **域名验证失败** - 检查 DNS 记录是否被修改或删除
2. **域名已过期** - 续费域名
3. **CA 机构问题** - 切换到其他 CA 重试
4. **速率限制** - 等待一段时间后重试

您可以在证书详情页查看续期历史和错误日志，也可以手动触发续期。

### 可以手动触发续期吗？

可以。在证书详情页面点击「立即续期」按钮即可手动触发续期，无需等待到期。

注意：频繁续期可能触发 CA 机构的速率限制。

### 续期后需要重新部署证书吗？

- **开启了自动部署**：无需操作，系统会自动部署新证书
- **手动部署**：需要重新下载证书并手动部署

建议配置自动部署，彻底实现证书管理自动化。

## 技术问题

### 支持 ACME v2 协议吗？

是的。anssl 基于 ACME v2 协议实现，兼容所有支持 ACME v2 的 CA 机构。

### 可以通过 API 管理证书吗？

可以。anssl 提供 RESTful API，支持：

- 申请证书
- 查询证书状态
- 下载证书
- 配置自动部署
- 域名监控

API 文档请访问控制台的「开发者中心」。

## 其他问题

### 证书可以用于内网环境吗？

Let's Encrypt 等免费 CA 只能为公网可访问的域名颁发证书。如果是内网环境：

- **公网域名 + 内网解析**：可以申请证书，使用 DNS 验证
- **纯内网域名**：需要自建 CA 或使用自签名证书

### 可以申请 IP 地址的证书吗？

目前 Let's Encrypt、ZeroSSL 等免费 CA 目前暂时不支持为 IP 地址颁发证书。

预计年底支持。

### 证书可以迁移到其他平台吗？

可以。证书是标准格式，下载后可以部署到任何支持 SSL/TLS 的平台，不绑定 anssl。
